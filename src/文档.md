# 山东麻将真实接口清单（以当前代码为准）

> 适用对象：前端联调 / 测试 / 产品确认  
> 结论先行：当前山东麻将后端**已具备完整联调链路**，请以前缀为 `/room` 与 `/mahjong/game` 的接口为准。  
> 本文以当前 controller 实现为准，旧文档中与本文冲突的路径，统一视为历史漂移。

## 2. 正式联调入口说明

### 2.1 正式房间开局唯一入口

前端如果是 4 人房间联机流程，**只能用**：

- `POST /room/{roomId}/start`

不要把 `POST /mahjong/game/init` 当成正式开局接口。  
`/mahjong/game/init` 只是独立调试入口。

### 2.2 房间绑定牌局不允许单独发牌

如果该牌局已绑定 `roomId`：

- 不允许再调用 `POST /mahjong/game/{gameId}/deal`
- 应改走 `POST /room/{roomId}/start`

## 3. 前端最小联调流程

### 3.1 房间流程

1. `POST /room/create`
2. `POST /room/{roomId}/join`
3. `POST /room/{roomId}/ready`
4. `POST /room/{roomId}/start`
5. `GET /room/{roomId}` 或 `GET /mahjong/game/{gameId}` 刷新状态

### 3.2 对局流程

1. `GET /mahjong/game/{gameId}/operations?playerId=xxx&seat=x`
2. 根据返回能力决定是否显示：出牌 / 摸牌 / 碰 / 吃 / 杠 / 胡 / 过
3. 调用对应动作接口：
   - `/discard`
   - `/draw`
   - `/pong`
   - `/chow`
   - `/kong`
   - `/win`
   - `/pass`
4. 牌局结束后读取：
   - `roundSettlement`
   - `/room/{roomId}/records/latest`
5. 下一局：
   - 全员再次 `POST /room/{roomId}/ready`
   - 房主再次 `POST /room/{roomId}/start`

## 6. 当前真实业务行为说明（前端必须知道）

### 6.1 新玩家加入后，会重置全房准备状态

当前 `joinRoom` 的真实逻辑是：

- 新玩家加入空座后
- 后端会执行 `resetReadyStates(room)`
- 即：**房间内所有玩家的 `ready` 都会被重置为 `false`**

这不是前端 bug，而是当前后端规则。

### 6.2 有人离开房间后，也会重置全房准备状态

当前 `leaveRoom` 同样会重置准备状态，并重置整场对局进度。

### 6.3 房间成员变化后，会重置整场 match 进度

当前实现中，加入 / 离开房间后还会执行 `resetMatchProgress(room)`，会清理：

- `matchId`
- `currentGameId`
- `lastFinishedGameId`
- `currentRoundIndex`
- `completedRoundCount`
- `currentDealerPosition`
- `continuousDealerCount`
- 座位累计分 / 胜局数 / 庄局数

也就是说，当前规则是：

> 房间成员发生变化，视为重新组织新的一场 match。

### 6.4 只有房主可以开始游戏

`POST /room/{roomId}/start` 只有房主可调用。

### 6.5 房间必须满 4 人且全员准备才能开始

否则后端会拒绝开始。

### 6.6 游戏进行中不能改准备状态 / 不能加入 / 不能离开

这是当前真实规则。

### 6.7 达到局数上限后，房间进入只读大结算状态

当 `completedRoundCount >= maxRoundCount`：

- `room.matchCompleted = true`
- 房间保留 `FINISHED`
- 不能继续 `ready/start`

- `/room/{roomId}/records/latest`

### 推荐优先级

- 操作能力查询：优先使用 `/mahjong/game/{gameId}/operations`
- 正式开局：只使用 `/room/{roomId}/start`
- 牌局刷新：优先使用 `GET /mahjong/game/{gameId}` + `GET /room/{roomId}`
